# Contributing

1. Review and ensure you meet the 3/10 test reproducible standard before submission.
2. Anonymize your log patterns.
3. Use the CLI `npx @korext/regression-submit draft` for interactive scaffolding.
4. Ensure your example correctly uses `js-yaml` and complies with the Spec schema.

All code modifications to the runner or DB require pulling via standard PR to main with complete tests added to the integration script layer.
